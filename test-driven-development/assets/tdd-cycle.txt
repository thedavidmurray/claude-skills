# TDD Red-Green-Refactor Cycle (ASCII Art)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         TDD CYCLE (MANDATORY)                           │
└─────────────────────────────────────────────────────────────────────────┘

  ┌──────────────┐
  │   START      │
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────┐
  │  1. RED              │
  │  Write failing test  │
  │  ✓ One behavior      │
  │  ✓ Clear name        │
  │  ✓ Real code         │
  └──────┬───────────────┘
         │
         ▼
  ┌──────────────────────┐
  │  2. VERIFY RED       │◄────────┐
  │  Run test            │         │
  │  Must fail!          │         │
  └──────┬───────────────┘         │
         │                         │
         │  ┌───────────────┐     │
         ├──┤ Passes? ERROR │─────┘
         │  └───────────────┘  Fix test
         │
         │  ┌───────────────┐     │
         ├──┤ Wrong reason? │─────┘
         │  └───────────────┘  Fix test/code
         │
         │ Fails correctly
         ▼
  ┌──────────────────────┐
  │  3. GREEN            │
  │  Minimal code        │
  │  Just pass test      │
  │  No extra features   │
  └──────┬───────────────┘
         │
         ▼
  ┌──────────────────────┐
  │  4. VERIFY GREEN     │◄────────┐
  │  Run ALL tests       │         │
  │  Must pass!          │         │
  └──────┬───────────────┘         │
         │                         │
         │  ┌───────────────┐     │
         ├──┤ Any fail?     │─────┘
         │  └───────────────┘  Fix code
         │
         │ All green
         ▼
  ┌──────────────────────┐
  │  5. REFACTOR         │
  │  Clean up            │
  │  Remove duplication  │
  │  Improve names       │
  │  Keep tests green!   │
  └──────┬───────────────┘
         │
         ▼
  ┌──────────────────────┐
  │  More features?      │
  └──────┬───────────────┘
         │
         ├─ Yes ──────────┐
         │                │
         ▼                │
  ┌──────────────┐        │
  │   COMPLETE   │        │
  └──────────────┘        │
                          │
                          └──► Back to step 1 (RED)


═══════════════════════════════════════════════════════════════════════════

CRITICAL GATES (Must pass before continuing):

  GATE 1: Before writing production code
  ┌────────────────────────────────────────┐
  │ Does a FAILING test exist for this?    │
  │                                        │
  │ NO  → STOP - Write test first          │
  │ YES → Continue to GREEN                │
  └────────────────────────────────────────┘

  GATE 2: Before claiming complete
  ┌────────────────────────────────────────┐
  │ Did you watch each test FAIL first?    │
  │ Did you watch each test PASS after?    │
  │ Are ALL tests green?                   │
  │ Is output pristine (no warnings)?      │
  │                                        │
  │ ANY NO → NOT complete                  │
  │ ALL YES → Complete                     │
  └────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════════════

DANGER ZONES (Delete code and start over):

  🚨 Code written before test
  🚨 Test passes immediately on first run
  🚨 Can't explain why test failed
  🚨 Skipped verification step
  🚨 "I'll test after"
  🚨 "Just this once"
  🚨 "Already manually tested it"
  🚨 "Deleting X hours is wasteful"

═══════════════════════════════════════════════════════════════════════════
```

## Example Session Transcript

```
You: "I need to add a retry mechanism for failed operations"

Good Response (TDD):
┌────────────────────────────────────────────────────────────────────
│ 1. I'll write a test first showing the retry behavior:
│
│    test('retries 3 times before failing', async () => {
│      let attempts = 0;
│      const operation = () => {
│        attempts++;
│        if (attempts < 3) throw new Error('fail');
│        return 'success';
│      };
│      const result = await retryOperation(operation);
│      expect(result).toBe('success');
│      expect(attempts).toBe(3);
│    });
│
│ 2. Running test... ❌ FAIL: retryOperation is not defined
│    ✓ Correct failure reason
│
│ 3. Minimal implementation:
│    [shows code]
│
│ 4. Running test... ✅ PASS
│    Running all tests... ✅ ALL PASS
│
│ 5. Refactoring... [if needed]
│
│ ✓ Complete
└────────────────────────────────────────────────────────────────────

Bad Response (NOT TDD):
┌────────────────────────────────────────────────────────────────────
│ Here's a retry implementation with exponential backoff:
│ [shows code without test]
│
│ You can test it like this:
│ [shows test after code]
│
│ 🚨 VIOLATION: Code before test
│ 🚨 ACTION REQUIRED: Delete code and start with test first
└────────────────────────────────────────────────────────────────────
```
